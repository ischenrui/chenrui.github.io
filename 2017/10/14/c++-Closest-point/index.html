最近点对问题

发表于 2017-10-14 | 分类于 C++

问题描述

给定平面上n个点，找其中的一对点，使得在n个点的所有点对中，该点对的距离最小。严格地说，最接近点对可能多于1对。为了简单起见，这里只限于找其中的一对。

解决思路

暴力搜索

最容易想到的就是遍历所有点对的距离，然后找出最小的点对距离。很明显这样做复杂度过高，对于n个点，需要计算 n-1 + n-2 + … + 1 次距离，复杂度为 O(n^2)。下面探寻一种利用分治法求解的方法。

分治法解决

这个问题很符合使用分治法的适用条件，即子问题相互独立且有最优子结构。假设二维平面有n个点，下面描述过程：

  1. 分别按X、Y轴对所有点进行排序（复杂度2*nlogn）
  2. （分解）将n个点以中间线划分为左右两个子集S1，S2
  3. 求出S1，S2两个子集中的最近点对 <Pl1,Pl2>，<Pr1,Pr2>，最近距离分别为D1,D2。但最近距离不一定为其中的最小值，因为还可能存在点对<Plx,Prx>的距离小于D1，D2。
  4. （合并）求出中间区间的最近点对<Plx,Prx>其距离为Dx，那么D1，D2，Dx中的最小值则为最近点对距离。

难点在于合并的时候如何在线性时间内完成，否则复杂度并不能因为采用分治法而降低。
这里只需要去检查宽度为 D = min(D1,D2) 的区间即可。
