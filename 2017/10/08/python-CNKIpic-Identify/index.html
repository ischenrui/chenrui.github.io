使用Python进行验证码识别

发表于 2017-10-08 | 分类于 Python

1. 问题描述

  • 目的：识别验证码图片中的数字或字母
  • 输入：验证码图片
  • 输出：字符串644J2

2. 解决思路

要识别图片中的字符，大致的处理方法一般是先进行图片字符切割，然后对单个字符进行模式匹配，最后把结果合并即可。这里涉及两个问题，一个是图像的处理，一个是字符串的模式匹配，故总体过程可以分为：

  1. 验证码图片的预处理
  2. 使用SVM模型进行识别字符

3. 验证码图片预处理

验证码图片的特点是像素低，掺杂干扰线或者斜体、旋转、部分重叠等等因素，可以说是千奇百怪。好在CNKI的验证码比较整齐，比较好处理。通用的办法大致是：

  1. 图片灰度化
  2. 图片二值化
  3. 图片去噪
  4. 图片的字符串切割
  5. 字符串图片尺寸标准化

3.1 图片灰度化

图片灰度化是指RGB图片的R=G=B，这样灰度图像每个像素只需一个字节存放灰度值，范围为0-255。大大减少了计算过程的复杂性。

3.4 图片的字符串切割

关于图片切割的算法有非常多，比如投影法，水滴法等等，没有仔细去了解，这里只介绍我选择的方法。
首先看了大多采用的方法是竖直统计黑色像素点，当像数量小于某个值(0,1,2…)的时候定义为切割线，这种方法很可能产生错误的切割，字符串的粘连、特殊字符串(M、H)都可能误切。

于是首先想到检测最左、最右像素确定含有字符的区域的宽度（高度都是恒定）
可以得到长度len,然后len/5即为每块儿字符的宽度

4. 使用SVM模型进行识别字符

SVM模型使用第三方包，参考LibSVM for Python 使用，处理过程为：

  1. 确定子图的特征值的定义
  2. 手动标记图标，训练SVM模型
  3. 使用训练好的SVM模型识别验证码图片

这一阶段的工作相对之前图片切割而言较为枯燥，调用SVM去识别也是使用性质，较为简单。

4.1 确定子图的特征值的定义

子图特征值定义可以每个像素点作为一个特征值，但明显复杂度过高，参考教程定义每行每列的黑色像素点数和作为特征值，13*12即25个特征值。
（这里我考虑过是否有必要和切割的时候一样，斜着统计数量，但应该差不不大，毕竟只是作为特征值反映图片的特征，同一类型的图片竖直统计也能反映图片特征）
